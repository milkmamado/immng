# 진단 카테고리 필드 변경 및 단기 치료 환자 관리 프로세스 개선 안내

**작성일**: 2025년 11월 13일  
**대상**: 부인과수술후회복 / 척추관절 환자 담당 매니저

---

## 📋 변경 개요

환자 관리 시스템의 진단 정보 처리 방식이 개선되었으며, **부인과 수술 후 회복** 및 **척추질환** 환자에 대한 자동 관리 상태 업데이트 로직이 추가되었습니다.

---

## 🔄 주요 변경사항

### 1. 진단 필드명 변경

**변경 전**: `diagnosis` (상세 진단명)  
**변경 후**: `diagnosis_category` (진단 카테고리)

#### 📍 영향 범위
- 일일 현황 그리드 (DailyStatusGrid)
- 환자 목록 표시
- 진단 정보 필터링

#### 💡 변경 이유
- 진단 정보를 카테고리 단위로 표준화하여 일관성 있는 관리 가능
- 단기 치료 환자 자동 분류를 위한 기반 마련

---

### 2. 단기 치료 환자 자동 관리 로직 추가

#### 🎯 대상 진단 카테고리
다음 진단을 받은 환자는 **단기 치료 환자**로 자동 분류됩니다:

1. **부인과 수술 후 회복**
2. **척추질환**

#### ⚙️ 자동 관리 프로세스

##### A. 경과 일수 계산 기준
시스템은 다음 우선순위로 마지막 활동일을 계산합니다:

1. **최우선**: `last_visit_date` (마지막 내원일)
2. **차선**: `inflow_date` (유입일)
3. **최종**: `created_at` (환자 등록일)

##### B. 자동 상태 업데이트 기준

| 경과 일수 | 자동 변경되는 관리 상태 | 의미 |
|----------|---------------------|------|
| **21일 이상** | 아웃 | 장기간 미접촉으로 이탈 간주 |
| **14-20일** | 아웃위기 | 곧 이탈 가능성 높음 (즉시 조치 필요) |
| **14일 미만** | 관리 중 | 정상적인 관리 범위 내 |

##### C. 자동 업데이트 제외 대상

다음 **최종 상태**의 환자는 자동으로 상태가 변경되지 **않습니다**:
- 사망
- 상태악화
- 치료종료

## 👥 담당 매니저 액션 아이템

### ✅ 즉시 조치 사항

#### 1. 진단 카테고리 확인 및 업데이트
- [ ] 담당 환자 중 진단 정보가 누락된 환자 확인
- [ ] "부인과 수술 후 회복" 또는 "척추질환" 해당 여부 명확히 기재
- [ ] 진단 카테고리 표준화 (기존 `diagnosis` → `diagnosis_category`)

#### 2. 단기 치료 환자 집중 관리
- [ ] 14일 이상 미접촉 환자 우선 확인
- [ ] "아웃위기" 상태 환자에 대한 긴급 연락 실시
- [ ] 재방문 일정 확정 시 시스템에 `last_visit_date` 즉시 업데이트

#### 3. 일일 체크 시스템 활용
- [ ] 매일 일일 현황 그리드에서 담당 환자 상태 확인
- [ ] 내원/연락 시 해당 날짜에 체크 기록 필수 입력
- [ ] 특이사항 발생 시 메모 기능 활용

---

## 🔔 시스템 자동화 프로세스

### 자동으로 처리되는 사항
1. ✅ 경과 일수 자동 계산
2. ✅ 관리 상태 자동 업데이트 (14일 기준)
3. ✅ 이탈 위험 환자 자동 분류

### 매니저가 직접 처리해야 하는 사항
1. ⚠️ 환자 연락 및 상담
2. ⚠️ 재방문 일정 조율
3. ⚠️ 일일 체크 기록 입력
4. ⚠️ 진단 카테고리 정보 입력

---

## 📊 모니터링 방법

### 일일 현황 페이지 활용
1. **진단 정보 확인**: 각 환자 카드에 진단 카테고리가 표시됩니다
2. **상태 뱃지 확인**: 
   - 🔴 빨강 = 아웃 (21일+)
   - 🟡 노랑 = 아웃위기 (14-20일)
   - 🟢 초록 = 관리 중 (14일 미만)
3. **필터 활용**: 진단명으로 검색하여 해당 환자만 필터링 가능

### 이탈 위험 환자 페이지
- "아웃" 및 "아웃위기" 환자 자동 집계
- 우선 조치 대상 리스트 확인 가능
